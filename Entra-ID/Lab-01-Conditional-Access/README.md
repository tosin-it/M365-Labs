# Lab 01 - Conditional Access Policy

## Objective

Create a Conditional Access policy in Microsoft Entra ID to require multifactor authentication for administrative accounts.

## Environment

- Microsoft 365 E5
- Microsoft Entra ID

## Configuration

- Created a Conditional Access policy
- Targeted Global Administrator role
- Applied policy to all cloud applications
- Configured MFA as the grant control
- Deployed initially in Report-only mode for validation

## Validation

Verified the policy configuration, assignments, and status in Microsoft Entra ID.

## Skills Demonstrated

- Microsoft Entra ID
- Conditional Access
- Zero Trust
- Identity security
- Multi-factor authentication (MFA)
